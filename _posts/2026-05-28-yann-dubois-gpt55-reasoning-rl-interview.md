## 1｜核心蒸馏

Yann Dubois 系统性拆解了2026年AI"突然好用了"的底层逻辑。核心论点：**进步是连续的，但可靠性跨过阈值后，用户的体感会出现阶跃**。他给出了三个结构性原因——可靠性临界点、自我加速循环、RL从竞赛走向真实世界——并贯穿全篇深入解释了GPT-5.5的设计哲学：效率提升、训练范式、泛化与持续学习。

## 2｜为什么2026年AI体感上"阶跃"了

**原因一：可靠性阈值（约2025年12月跨过）**

Dubois 说，在此之前模型做不了真实工作——不是能力不够，而是信不过。每一步都有小概率出错，agentic 模型跑得越久，最终整体正确的概率越低。OpenAI 的工作重点是**降低每一步的出错概率**，当这个概率降到某个临界值以下，用户体验就从"偶尔惊艳"变成了"确实好用"。

**原因二：自我加速反馈循环**

模型好用了 → 全员用模型加速开发 → 更快迭代下一轮。Dubois 强调内部"每个人都在用自家模型做开发"，工具链本身就是模型训练的一部分。

**原因三：RL 从竞赛走向真实世界**

o1/o3 时代的 RL 优化的是"可验证奖励"——数学竞赛、编程竞赛——有明确的答案对错。现在这个工具箱被扩展到了**真实世界的模糊任务**。模型不再只是优化比赛得分，而是优化"给用户提供的实际效用"。

## 3｜GPT-5.5 的技术核心

**效率提升：整体约2倍加速**

来自两个独立方向的叠加：
- **推理优化**（工程侧）：更快生成每个 token
- **"思考更少"**（研究侧）：将 test-time scaling curve 左移——用更少的思考 token 达到同样的精度

Dubois 用了一个类比：专家 vs 本科生的区别。专家知道哪些路径大概率是错的，根本不会去探索；模型通过 RL 被训练成"学会早期识别错误路径并回溯"。

**Thinking vs Pro**

纯粹是 test-time compute 的权衡：
- **Thinking**：默认模式，日常使用
- **Pro**：将 scaling curve 向右延伸，用更多 token 换取更高的正确概率，适合数学家在后台跑一两个小时的场景

**内部团队结构**

- **纵向团队**：编码、科学、安全、计算机使用等特定用例各自攻坚
- **横向团队（Dubois 的 Post-training Frontiers）**：指令遵循、思考时间、函数调用、记忆——跨所有领域通用的改进
- Dubois 团队的角色是**"平滑函数"**——把纵向改进整合成连贯的最终模型

## 4｜三段训练管线：预训练 → 中训练 → 后训练

这是 Dubois 对整个现代 AI 训练范式的精炼概括：

| 阶段 | 比喻 | 核心任务 |
|------|------|----------|
| **预训练** | 进图书馆读所有书 | 从互联网学习一切知识 |
| **中训练** | 重点读经典教材 | 过采样高质量数据（Wikipedia、GitHub），压制噪声 |
| **后训练** | 从读书人变成专家 | SFT + RL，把"什么都知道的模型"变成"对用户有用的工具" |

**关键洞察**：越靠近末端的阶段，迭代周期越短（从月级到天级）。这也是为什么后训练（post-training）成为当前最重要的研究前沿——你可以快速实验、快速验证。

**SFT vs RL**：
- SFT（监督微调=行为克隆）只能克隆人类表现，无法超越
- RL（强化学习）优化奖赏函数，可以超越人类标注者能力上限

## 5｜RL：从"太脆弱"到"突然好用"

Dubois 自曝曾是 RL 怀疑论者——在斯坦福做 Alpaca 时，他是试图**不用 RL 复现 ChatGPT**。当时他和 Yann LeCun 一样，认为 RL 只是"蛋糕上的樱桃"。

**转折点**：当模型规模跨过一定阈值（具备"好的先验"），RL 突然开始有效。Dubois 认为机器人领域也在经历同样的过程。

**RL 的三大难题**：
1. **基础设施成本**：需要大规模采样 → 极其昂贵
2. **长 rollout 的 credit attribution**：agentic 系统运行完才知道对错，难以判断具体哪一步有效
3. **RL 依赖先验质量**：模型底子差，RL 无法生效

**开源社区的收敛**：GRPO 已成为主流——最简单、最可扩展的方法。ML 的经验一再证明：**最简单、能最多堆算力的方法最终胜出**。

## 6｜幻觉的机制性理解

> **SFT 可以导致幻觉，RL 可以抑制幻觉**

Dubois 引用 John Schulman 的洞见：
- **SFT 场景**：模型不知道某篇论文，但人类标注答案里提供了引用 → 模型被训练去 citing 它不知道的东西 → 被奖励编造
- **RL 场景**：模型从自己的分布中采样，不可能偶然猜对不知道的内容 → 错了就被惩罚 → 消除编造行为

这是当前对幻觉最为机制性的理解之一。

## 7｜泛化的三种层次

Dubois 区分了三个不同的泛化维度：

1. **算法泛化**（✅ 已解决）—— GRPO 在多个领域都能直接应用
2. **能力泛化**（✅ 部分解决）—— C++ 训练好 → 其他编程语言也受益
3. **水平泛化**（❌ 未解决）—— 现实世界的问题"未充分定义"（underspecified）。数学竞赛所有信息都在 prompt 里，但真实场景需要自己去互联网找资源。这种"导航模糊性"的能力模型还很欠缺

**负泛化现象**：精确的指令遵循反而可能损害隐性理解——例如用户打错了文件名，模型会精确执行错误的指令，而人类会意识到这是个笔误。

## 8｜持续学习——AI 最大的未解决问题之一

Dubois 提供了一个简洁但深刻的框架：

- **T0 时刻**：现有模型比大多数新员工起点高
- **随时间推移**：人类学习曲线陡峭上升，模型基本持平
- **关键指标**：**"曲线下面积"**——人类在长期使用中累积效用更高

> "我其实很惊讶这个问题 ChatGPT 发布三年后还没解决。我不完全理解为什么花了这么久。"

他认为这是一个资源投入问题——如果全力投入，理论上能解决。但隐私、权限等工程挑战使问题复杂化。

## 9｜对初创公司和从业者的建议

> **"瓶颈不是模型能力，是最后一英里"**

- OpenAI 会持续专注**横向通用能力**
- 留给垂直领域的空间巨大——**权限、连接器、领域特定工作流**才是真正的价值所在
- Agent harness 是**短期有用**（80% → 85% 可靠性），但通用 harness 不会长期可持续
- "如果今天冻结模型能力，全力优化最后一英里，人们已经能在每个领域感受到 AGI 级别的影响"

## 10｜值得关注的其他洞察

- **预训练没有撞墙**：更大模型反而更高效，因为思考需要的 token 更少，GPU 推理并行度更高
- **多模态 vs 体感智能**：Anthropic 证明文本为主也能很强，但具身 AI 会带来常识突破
- **模型即裁判**：模型-as-judge 创造了自增强的能力飞轮，是评估领域最重要的进展
- **GDPval 和真实评估**：评估正在从静态 benchmark 转向模拟真实经济活动的评测
- **Craft → Science 循环**：AI 建设总是先炼金术再科学化，这个循环在管线中不断重复
