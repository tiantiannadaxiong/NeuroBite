## TL;DR｜核心结论速览

> 1. AI 进步是连续的，但**可靠性跨过阈值（2025年12月）**后，用户体感出现阶跃——这是"突然好用了"的根本原因，而非模型能力突变。
>
> 2. **RL 从可验证奖励（数学/编程竞赛）扩展到模糊真实场景**，是当前能力提升最关键的推动力。Dubois 自述曾是 RL 怀疑论者，承认"跨过规模阈值后 RL 突然开始有效"。
>
> 3. **持续学习是 AI 最大的未解决问题之一**——模型 T0 起点高但曲线平坦，人类学习曲线陡峭，\"曲线下面积\"使人类在多数场景仍更有效。ChatGPT 发布三年后仍未解决。

## 1｜核心观点（Core Views）

**观点 1：进步体感的阶跃来源于可靠性临界点，而非能力突变**

- **支撑逻辑**：Agentic 模型每一步都有小概率出错，持续时间越长，最终正确概率越低。当每步错误概率降到某个阈值以下，用户从\"不敢信任\"变为\"可以托付工作\"——这种体感变化是阶跃的，但底层进步是连续的。
- **声明类型**：`[第一手]`——来自团队内部观测
- **可信度判断**：逻辑自洽，与客观的使用体感变化时间线一致。可视为可靠的第一手判断。

**观点 2：RL 在跨过规模阈值后从\"太脆弱\"变成\"突然有效\"**

- **支撑逻辑**：模型具备\"好的先验\"（good priors），RL 才开始工作。Dubois 自述曾是 RL 怀疑论者——在 Alpaca 试图不用 RL 复现 ChatGPT。开源社区已收敛到 GRPO，最简单、最可扩展的方法胜出。机器人领域正在经历同样的阶段。
- **声明类型**：`[第一手]`——亲自经历从怀疑到使用
- **可信度判断**：高。Dubois 对 Alpaca 的坦诚说明（试图绕开 RL）增强了可信度。GRPO 胜出符合 ML 历史经验（最简可扩展方法最终胜出）。

**观点 3：SFT 实际上可以导致幻觉，RL 可以抑制幻觉**

- **支撑逻辑**：SFT 中模型被训练去复制包含它不知道的信息的答案（如人类标注者提供的引用）→ 被奖励编造。RL 中模型从自身分布采样，不可能偶然猜对未知内容 → 错了就被惩罚 → 编造行为消失。
- **声明类型**：`[第一手]`——引用 John Schulman 的成果，Dubois 团队内部验证
- **可信度判断**：逻辑严密，机制清晰。这是对幻觉问题最机制化的解释之一。

**观点 4：持续学习是三年未解决的最大瓶颈之一**

- **支撑逻辑**：框架——T0 基线模型 > 新员工，但人类学习曲线陡峭，模型曲线平坦。关键竞争指标是\"曲线下面积\"（累积效用）。Dubois 坦言\"不完全理解为什么花了这么久\"。
- **声明类型**：`[第一手]` + `[推断]`
- **可信度判断**：框架本身有力。Dubois 坦诚不知道原因的态度比给出一个经不起推敲的解释更可信。

**观点 5：瓶颈是最后一英里，不是模型能力**

- **支撑逻辑**：即使冻结当前模型能力、全力优化 harness（权限、连接器、垂直工作流），用户已能在每个领域感受到 AGI 级别的影响。OpenAI 专注横向通用能力，留给垂直创业公司的空间巨大。
- **声明类型**：`[个人判断]`——但基于一手经验
- **可信度判断**：逻辑合理，且 Dubois 在此有利益关系（鼓励生态建设），但判断本身与外部观测一致。

## 2｜话题分析（Topic Breakdown）

### 话题 1：GPT-5.5 发布与内部情绪周期

- **核心信息**：每个重磅模型都要经历"兴奋 → 怀疑 → 低估 → 满意"的情绪周期。5.5 的振幅更大，但最终外部反响好。迭代周期从月级（预训练）到天级（后训练）。
- **关键细节**：效率提升 ~2x（推理优化 + 模型思考更少）；Pro 模型是 test-time scaling curve 向右延伸；Thinking vs Pro 仅取决于投多少 test-time compute。
- **值得注意**：Dubois 对内部情绪周期的描述非常具体，暗示团队管理是模型质量的重要变量。

### 话题 2：三段训练管线（预训练 → 中训练 → 后训练）

- **核心信息**：预训练（学一切）→ 中训练（过采样高质量数据）→ 后训练（SFT 行为克隆 + RL 超越人类）。越靠近末端迭代越快。
- **关键细节**：Alpaca 时代仅用 ~50K SFT 样本，Kimi / DeepSeek 在 RL 阶段已使用 ~百万级数据点。预训练没有撞墙——更大模型反而更高效（思考 token 更少 + GPU 并行更优）。
- **值得注意**：Dubois 间接确认了 Anthropic 模型的\"规模明显更大\"（根据 token 定价推断）。

### 话题 3：模型泛化的三个层次

- **核心信息**：算法泛化（GRPO 跨领域好用 ✅）> 能力泛化（C++ 好 → 其他编程语言也好 ✅）> 水平泛化（处理未充分定义任务的能力 ❌）。数学竞赛中的信息都在 prompt 里，但真实场景需要自己去搜集和理解上下文。
- **关键细节**：\"负泛化\"现象——精确指令遵循损害隐性理解（如打错文件名模型仍执行）。水平能力的缺失是**跨领域**的——模型在某领域不知道就说不知道，它在所有领域都这样。
- **值得注意**：这是最值得深入阅读的技术判断之一。水平泛化的瓶颈可能是通向通用智能的关键约束。

### 话题 4：评估危机与模型即裁判

- **核心信息**：模型越强，评估越难——任务更开放、正确答案不唯一。模型超越多数人类，造成评估者短缺。模型-as-judge 形成自增强能力飞轮。
- **关键细节**：Dubois 加入 OpenAI 后第一个项目主动选择了\"数据和评估\"——因为这是最被低估的方向。
- **值得注意**：评估与训练数据之间存在\"泄露\"——用某个 eval 得到的训练数据再去优化，eval 很快失效。

### 话题 5：Agent Harness 与创业机会

- **核心信息**：通用 harness 不可持续，垂直 harness 是必要的短期策略。瓶颈始终是\"最后一英里\"——权限、连接器、领域知识。
- **关键细节**：即使冻结现有模型 + 全力优化 harness，用户已经能在每个领域感受到 AGI 级别的影响。但因为模型还在快速进步，最终形态的 harness 尚未可知。
- **值得注意**：⚠️ Dubois 在此有利益关系（鼓励生态而非与 OpenAI 竞争），但其判断的逻辑自洽。

**提问者策略（附）**：Matt Turck 的追问质量高——对每个核心观点都要求给出具体机制（\"How does it work?\"），迫使 Dubois 从宏观论断下降到具体解释。在持续学习和模型能力瓶颈问题上持续追问，获得了\"我不完全知道\"这种有价值的坦诚回应。

## 6｜可操作信息（Actionable Takeaways）

1. **RL 方法的收敛（GRPO）意味着开源生态已找到实质可行的后训练路径**——如果你使用开源模型且关心推理能力，GRPO 是当前风险最低的选择。

2. **\"最后一英里\"是真实瓶颈——短期可操作的方向是垂直领域 harness + RL fine-tuning**。知识图谱、权限管理、连接器、领域特定奖励函数——这些是当前 ROI 最高的投入方向。

3. **持续学习是未开采的矿脉**——如果 OpenAI 三年未解决，说明这个问题需要的基础创新远超预期。关注这个方向的研究突破，可能带来结构性机会。

4. **评估短缺正在升级为系统性瓶颈**——模型超越人类评估者的速度超过预期。\"模型即裁判\"的能力飞轮在加速，但其自我增强偏差的风险尚未被充分讨论。

5. **可靠性阈值作为 AI 部署决策框架**——不必等待模型能力大幅提升，关注\"单步错误率是否已降至可靠工作的临界值\"。对垂直应用来说，这个阈值可能已经被跨过。

---

## 7｜一句话总结

> OpenAI 后训练的一线负责人确认：2026年的体感阶跃并非能力突变，而是可靠性跨过阈值 + RL 从竞赛走向真实 + 自我加速循环三者共振的结果；瓶颈不在模型能力，在持续学习和最后一英里——这两个问题都远未解决。
